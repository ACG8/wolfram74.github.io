From 0 to Basic Games

Fundamental Javascript skills to make games

Over View

It is a common desire to make games, after all, playing games is one of the fundamental things humans do, and a game has to be invented before it can be played. People do it with friends all the time, while bored in the car or with a deck of cards. The rules are explained briefly and everybody just kind of runs with it. Computers are kind pretty bad at ambiguity though, so to make a game that a computer can run for people and take care of all the details, we have to be very explicit about what's going on. This is explicitness is handled by programming languages, we give instructions to the computer and it takes it from there. To make a game in a way that can be handled by a programming language we'll need to start breaking games down into more basic parts and figure out what skills allow us to deal with those parts. Now as a caveat, games are a pretty wide area of human thought, so any set of criteria specific enough to be a useful starting point is also likely going to be so specific as to rule some things out that are often referred to as games. Finding the short coming in basic descriptions is what we do after our skills have grown past the basics, so we'll put those details aside for now.

Components of a Game.

The following broad components make up games and using the classic game pong to provide an example.

  • STATE: a collection of information
  • MECHANICS: a collection of rules that dictate how STATE changes
  • CONTROLS: a collection of methods for a user to change STATE
  • INTERFACE: a means for a user or player to get feedback on what the STATE is
In Pong, the location of the ball and the players' paddles, the score of the players, this makes up the STATE of the game. The fact that the ball bounces off the walls and paddles, that a player score increases when the ball gets behind another players paddle, these are MECHANICS. That the players can move the paddles by key presses or moving the mouse are CONTROLS. The visual representation of where the paddles and the ball is the INTERFACE. Interface and graphics are often interchangeable, but not necessarily, many early games had a purely text based interface, so it is important understand that STATE and INTERFACE are different parts of a game.

Basic Skills related to STATE

State in a computer program is data stored in variables. A given piece of data is of a particular type, different languages support different types by default. Javascript's basic types include Number, String, Boolean, Array and Object.

  • Numbers you are most likely familiar with, things like 1, 4.3, -27.
  • Strings are any collection of letters and characters, denoted as being between single or double quotes, 'one string', "another string"
  • Booleans are just true and false, but they are used in for many parts of program flow.
  • Arrays are ordered collections of other kinds of data, like several numbers, several strings, or mixtures of both.
  • Objects are labeled collections of other kinds of data, a number labeled age and a string labeled name that are associated together, for example.

Advance Skills related to STATE

It is common that in making a game, some things occur multiple times. Take, for example, a Goomba in mario brothers. Each goomba would have the same kinds of state, but different values. Javascript allows you to, in effect, make your own data types. Creating your own data types involves writing a constructor function and running it prefixed with the 'new' keyword. It is a nuanced subject for people new to programming, but allows you to make your code much easier to conceptualize and work with. These datatypes you define are sometimes referred to as prototypes. W3Schools Javascript prototypes, and Guide to Javascript prototypes

Basic skills related to MECHANICS

Mechanics describe how a game responds to different circumstances, if mario runs into a turtle from the side, he takes damage, if he falls from the top the turtle is knocked out of it's shell. If Kerrigan tries to cast psionic storm with enough energy, she succeeds, loses some energy and enemies lose health, otherwise she fails and nothing happens. These are explicit rules about how state changes. In Javascript, functions are collections of code that modify or create data. They are run( or called) with input data (or arguments). One simple example of a function is from the Math library, Math.pow(2, 3) returns the number 8, because that's 2 to the 3rd power. You can write your own functions to do almost anything. The simplest decisions that happen in functions are like I described with Kerrigan, something is checked to be either true or false. If true, a thing happens. This is called an if-statement. If you wanted something else to happen, you would follow it with an else-statement. Introduction to functions in Javascript, and MDN Javascript Guide

Advanced skills related to MECHANICS

Functions that accomplish particular abstract goals are sometimes referred to as algorithms. For example, if you had many different players, but wanted to order them based on score, you would use a sorting algorithm to order them. If you had a non-player character decide it's own path through a map, it would use a path finding algorithm. Learning about algorithm design will help you to make complicated mechanics in games.
